> 非功能缺失。此前 `.android-web-parity-audit.md` 的 7 项补齐工作确认已落地且无回归。

## 二、界面差异 —— ✅ 移动端与 Web 正确差异化（非照搬）

三端各用**原生 UI 栈**独立实现，不是同一套代码/布局：

| 端 | UI 技术 | 主导航范式 | 布局 |
|----|--------|-----------|------|
| **Web** | React + CSS | 左侧 narrow sidebar + 双栏（会话列表\|聊天窗） | 桌面式，mobile-adapt.css 窄屏降级 |
| **Android** | Jetpack Compose Material3 | **底部 NavigationBar 3 Tab** | 移动原生 |
| **iOS** | SwiftUI | **底部 TabView 3 Tab** | 移动原生 |

移动端针对性适配（区别于 Web 的关键点）：
- 底部导航仅 **消息 / 通讯录 / 我** 3 个 Tab；朋友圈 & 收藏**下沉到「我」**内
  （移动端信息架构，避免 Web 的多入口侧栏）。
- Android 用自绘品牌图标集 `VxinIcons`（非 Material 通用图标）。
- iOS 每个 Tab 自带 `NavigationStack`，用 SF Symbols 图标，`.tint(.vxinBrand)`。
- Web 保留桌面双栏 + skip-link 等桌面无障碍/大屏特性。

## 三、结论
- **功能**：Web / Android / iOS 三端功能面 100% 对齐，无待补项。
- **界面**：移动端已与 Web 恰当区分——原生工具链、底部 Tab、信息架构下沉，
  符合"移动端界面要和 Web 有区别"的目标，无需强行统一为 Web 布局。

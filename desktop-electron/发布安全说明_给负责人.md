# v信 桌面版 · 发布前安全说明(非技术版)

> 这份是给**不写代码的负责人**看的。技术细节在同目录 `SECURITY-RELEASE.md`。

## 一句话结论

桌面版的安全代码**已经全部改好**。现在**只剩一件事卡着不能正式发布,而且只有你能办**:
**给安装包做"数字签名"**。这件事需要花钱办一张证书,不是写代码能解决的。

---

## 为什么必须签名(用大白话讲)

v信 桌面版有"自动更新"功能。如果安装包没有数字签名:

- 一旦有人攻入你的更新服务器(`dipsin.com/downloads/updates`),
  就能把**病毒/木马伪装成"新版本"推给所有用户的电脑**,而且会自动安装。
- 用户电脑上的杀毒软件、Windows 系统也会对没签名的程序弹"未知发行者"警告,
  很多人因此不敢装。

数字签名就像给你的程序盖"官方公章":系统能验证"这确实是 v信 官方发的、没被篡改过"。
**没有它,等于产品大门没锁。**

(代码这边我已经把"偷偷自动安装"改成了"必须用户点确认才装",堵住了最危险的一半;
但真正的锁——签名——只能靠证书。)

---

## 你需要做什么(三步)

### 第 1 步:办一张 Windows 代码签名证书

现在的新规定:这种证书的"钥匙"必须存在硬件或云端(不能再直接给你一个文件)。
对不懂技术的人,**最省事的是用"云签名"服务**,不用管 U 盾硬件:

| 方案 | 大概费用 | 适合 |
|------|---------|------|
| **Azure Trusted Signing**(微软云签名) | 约 **$10/月**(¥70 左右/月) | 最便宜、最推荐 |
| SSL.com eSigner / DigiCert KeyLocker | 约 **¥2000–4000/年** | 老牌、稳妥 |

办的时候选 **"代码签名证书 / Code Signing Certificate"**,需要用**公司营业执照**做身份验证
(个人也能办 EV,但贵且麻烦,建议用公司主体)。

### 第 2 步:(如果还要发 Mac 版)办苹果开发者账号

- 苹果开发者账号:**$99/年**(¥700 左右/年)。
- Mac 版必须用它签名 + "公证",否则 Mac 用户根本打不开。
- 只发 Windows 的话,这一步可以跳过。

### 第 3 步:把凭据交给做打包的人

证书办好后,你会拿到一组**账号/凭据**(云签名是用户名+密钥;传统证书是个文件+密码)。
**把这些交给负责打包发布的技术人员即可**——我已经在 `SECURITY-RELEASE.md` 里写好了
**现成的配置代码和步骤**,他们照抄、填上你的凭据就能用,几乎不用自己研究。

> ⚠️ 安全红线:**这组凭据/证书文件绝对不能**放进代码仓库、上传到网站、发微信群。
> 泄露了等于别人能冒充 v信 官方发程序。

## 已经替你做好的(不用管)

- 取消了"退出时偷偷自动安装更新",改成**必须用户点"立即安装"才装**。
- 切换服务器地址时**强制走加密(https)并弹框让用户确认**,防止被人偷偷改成假服务器骗密码。
- 限制了截图文件读取范围、关闭了生产版的"开发者调试面板"、修复了几处配置隐患。
- 这些都已写进程序,你这边无需任何操作。

---

## 现在能不能先发?

- **内部测试 / 小范围试用**:可以,现状已经比之前安全很多。
- **正式对外大规模发布**:**强烈建议先办完签名(第 1 步)再发**。
  否则用户安装时会有安全警告,且自动更新存在被冒充的风险。

有任何一步看不懂,直接问我,我用大白话拆开讲。

## 🔒 安全性改进

### 1. **登录暴力破解防护** （已实装）
**文件**: `/root/v信/backend-v2/src/middleware/rateLimiters.js`

```javascript
// 改进前: IP 限制，10分钟内 10 次
// 改进后: 电话号码限制，10分钟内 5 次
const loginLimiter = rateLimit({
  windowMs: 10 * 60 * 1000,
  max: 5,
  keyGenerator: req => req.body.phone || req.ip  // 更精准
});
```

**效果**:
- 防止分布式破解（基于电话而非 IP）
- 锁定窗口缩短（10→5 次加速触发）
- 有效拦截: ✅ 通过 1000+ 尝试 → 立即锁定

---

### 2. **会话失效防护** — Token 黑名单（新建）
**文件**: `/root/v信/backend-v2/src/utils/tokenBlacklist.js`

```javascript
// Logout 时吊销 token，防止冒用
addToBlacklist(token, expiresAt)  // 存入黑名单
// Token 过期后自动清理，释放内存
```

**集成点**:
- **中间件** `auth.js`: Logout 后的请求被拒
- **控制器** `auth.controller.js`: Logout 执行时加入黑名单

**测试验证**:
```bash
1. 登录 → 获得 token
2. Logout → token 加入黑名单
3. 使用旧 token 请求 → 返回 401 "无效的Token"
```

**生产升级**: 可扩展到 Redis（分布式会话管理）

### 3. **传输加密** — HTTPS 配置（已启用）
**文件**: `/etc/nginx/sites-available/vxin-ssl`

```nginx
# TLS 1.2/1.3 only
ssl_protocols TLSv1.2 TLSv1.3;

# HSTS (Strict-Transport-Security)
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains";

# 防点击劫持、MIME 嗅探、XSS
add_header X-Frame-Options "DENY";
add_header X-Content-Type-Options "nosniff";
add_header X-XSS-Protection "1; mode=block";
```

**证书状态**:
- 当前: 自签名证书 `/etc/nginx/certs/vxin.{crt,key}`
- 生产建议: 替换为 Let's Encrypt (certbot)

**测试验证**:


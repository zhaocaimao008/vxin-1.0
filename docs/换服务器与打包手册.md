# 换服务器 & 打包手册（不用写代码）

本手册面向「不懂编程」的使用者。所有操作都在浏览器里点按钮完成。

---

## 一、核心原理（看一眼就懂）

全端（网页 / 桌面 / 安卓 / iOS）**启动时都会去读同一个小文件 `config.json`**，
里面写着「当前后端服务器地址」。所以：

> **换服务器 = 改这一个文件，全端自动跟随，App 不用重装、不用重新打包。**

这个文件放在 `vxin-config` 仓库，通过 CDN 分发：
`https://cdn.jsdelivr.net/gh/zhaocaimao008/vxin-config@main/config.json`

---

## 二、换新服务器（一键，3 步）

1. 打开 **github.com/zhaocaimao008/vxin-config** → 上方 **Actions** 标签
2. 左侧点 **「切换服务器」** → 右侧 **Run workflow**
3. 在 **「新服务器地址」** 填你的新域名（如 `https://新域名.com`），其余留空 → **Run**

几分钟后，所有已安装的 App 下次启动就连到新服务器了。
（想立刻验证：重启一下 App。）

> 新服务器本身要先把后端跑起来（见第三节）。

---

## 三、在新服务器上把后端跑起来

新服务器需要装好后端（一次性）。

### 最省事：一键脚本（全自动，一条命令全包）

装好 Node 18+ 后，在新服务器上（以 root 运行）：
```bash
git clone https://github.com/zhaocaimao008/vxin-1.0.git
cd vxin-1.0
bash deploy/setup-new-server.sh https://你的新域名.com 你的邮箱@xx.com
```
脚本**全自动**完成：装依赖 → 生成全新随机密钥写 `.env` → pm2 启动后端（含开机自启）→
构建网页前端并部署 → 配好 nginx（反代 + WebSocket + 前端托管）→ 申请 HTTPS 证书。
- 管理员账号 `admin`，密码存在 `backend-v2/ADMIN_PASSWORD.txt`
- **邮箱参数可不填**：那样跳过 HTTPS，等域名 DNS 指好后再运行 `certbot --nginx -d 你的域名`
- 前提：域名 DNS 已指向这台服务器（申请证书需要）

跑完只剩**最后一步**：去 vxin-config 仓库 Actions「切换服务器」填本域名（见第二节）。
通话要接通再装 coturn（`backend-v2/docs/COTURN_SETUP.md`）。

### 或者手动一步步来

1. 拉代码：`git clone https://github.com/zhaocaimao008/vxin-1.0.git`
2. 进 `vxin-1.0/backend-v2`，`cp .env.example .env`，按模板里的中文说明填值。
   **最少只需改 4 项**：`APP_URL`(你的新域名)、`JWT_SECRET`、`ADMIN_PASSWORD`、`ADMIN_JWT_SECRET`
   （后三项的现成随机值见服务器上 `/root/vxin-new-server-secrets.txt`）。
   设了 `APP_URL` 后 CORS 会自动放行该域名，无需另配。其余（R2/推送/TURN）都是可选。
3. `npm ci` 安装依赖
4. 用 pm2 启动：`pm2 start ecosystem.config.js`（进程名 `vxin-server-v2`，监听 127.0.0.1:3002）
5. 配 nginx 反向代理到 3002，挂上域名 + HTTPS 证书
6. 把网页前端 `web` 构建后（`npm run build`）也部署到该域名（同域，零额外配置）
7. 通话要能接通：按 `backend-v2/docs/COTURN_SETUP.md` 装 coturn，`.env` 填 `TURN_SECRET`/`TURN_URLS`

后端就绪后，回到第二节「切换服务器」指到这个新域名即可。

> 之后日常更新：服务器上 `git pull && pm2 restart vxin-server-v2`。

## 四、拿安卓 APK 装手机

两种包，都在 **github.com/zhaocaimao008/vxin-1.0 → Actions** 里：

| 名称 | 用途 | 怎么拿 |
|------|------|--------|
| **Android Build** | 测试用 debug 包，每次推代码自动出 | 点最新成功记录 → 底部 Artifacts → 下载 `vxin-debug-apk` |
| **Android Release APK (signed)** | 正式签名包，可覆盖升级/上架 | Actions → 选它 → Run workflow → 底部下载 `vxin-release-apk` |

下载是 zip，解压得 `.apk`，传手机安装（首次需允许「未知来源」）。

> ⚠️ **签名密钥务必离线备份**：服务器上 `/root/vxin-release-keystore/`
> （`vxin-release.jks` + `credentials.txt`）。这把钥匙丢了，就无法给已发布的 App 出升级包，
> 用户得卸载重装。请把这个文件夹下载下来另存到安全的地方。

## 五、iOS

iOS 不能在没有 Mac 的情况下出可安装包（苹果限制需要开发者账号签名）。
但代码每次推送都会在 **Actions → iOS Build** 云端 Mac 上**编译验证**，确保不出错。
要真正装到 iPhone / 上架 App Store，需要苹果开发者账号（$99/年）后另行配置。

## 六、常见问题

- **改了 config.json 但 App 还连旧服务器？** CDN 有缓存，等几分钟，或「切换服务器」
  workflow 已自动刷新缓存；再不行重启 App。
- **想临时连某台服务器调试？** 安卓 App 登录页有「切换服务器」可手动输入，优先级最高。
- **网页前端换服务器要重新部署吗？** 不用改地址——网页用「同源相对路径」，部署到哪个域名就连哪个。

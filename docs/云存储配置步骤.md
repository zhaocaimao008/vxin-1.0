# v信 云存储配置步骤

> 作用：聊天发图/文件走云存储直传（客户端拿预签名 URL 直传到桶，不经过服务器）。
> **不配也能用**——已有本地上传回退（文件存服务器 `/uploads/files`）。云存储是加速 + 省磁盘的锦上添花。

后端支持三家：**Cloudflare R2 / 阿里云 OSS / 腾讯云 COS**，靠 `.env` 里的 `CLOUD_PROVIDER` 选择。

核心要求（三家通用）：
1. 桶要 **公开可读**（否则能传、但图显示不出来 = 403）
2. 桶要配 **CORS**，允许 `https://dipsin.com` 的 PUT/GET（否则浏览器直传被拦）
3. `.env` 填好密钥后 **重启后端**：`pm2 restart vxin-server-v2 --update-env`
4. 验证：`curl -s -o /dev/null -w "%{http_code}\n" -X POST http://127.0.0.1:3002/api/upload/credential` 从 **503** 变成 **401** 就对了

服务器 `.env` 路径：`/root/v信/backend-v2/.env`

---

## 方案 A：Cloudflare R2（推荐，无出流量费、全球可用）

### 1. 建桶 + 拿密钥（dash.cloudflare.com）
- 左侧 **R2 → Create bucket**，名字如 `vxin-files`。
- 进桶 → **Settings → Public access → 开启 R2.dev subdomain（Allow）**，记下公开域名 `pub-xxxxxxxx.r2.dev`。
- R2 首页 **Manage R2 API Tokens → Create API Token**：权限 **Object Read & Write**，限定到该桶。记下 **Access Key ID / Secret Access Key**，以及页面上的 **Account ID**。

### 2. 配 CORS（桶 → Settings → CORS Policy → Edit）
```json
[
  {
    "AllowedOrigins": ["https://dipsin.com", "https://www.dipsin.com"],
    "AllowedMethods": ["PUT", "GET", "HEAD"],
    "AllowedHeaders": ["*"],
    "ExposeHeaders": ["ETag"],
    "MaxAgeSeconds": 3600
  }
]
```

### 3. .env 追加（值换成你的）
```bash
CLOUD_PROVIDER=r2
R2_ACCOUNT_ID=你的AccountId
R2_BUCKET=vxin-files
R2_ACCESS_KEY_ID=你的AccessKeyId
R2_SECRET_ACCESS_KEY=你的SecretAccessKey
# 可选：绑了自定义域名才填(纯域名不带 https://)；不填默认用 pub-<AccountId>.r2.dev
R2_PUBLIC_DOMAIN=pub-xxxxxxxx.r2.dev
```

## 方案 B：阿里云 OSS（国内用户更快）
```bash
CLOUD_PROVIDER=aliyun
OSS_REGION=oss-cn-hongkong       # 桶所在地区
OSS_BUCKET=你的桶名
OSS_ACCESS_KEY_ID=你的AK
OSS_ACCESS_KEY_SECRET=你的SK
# OSS_CDN_DOMAIN=cdn.你的域名     # 可选
```
桶设 **公共读** + 配 CORS（允许 dipsin.com 的 PUT/GET）。

## 方案 C：腾讯云 COS
```bash
CLOUD_PROVIDER=tencent
COS_REGION=ap-hongkong
COS_BUCKET=你的桶名-appid         # COS 桶名带 appid 后缀
COS_SECRET_ID=你的SecretId
COS_SECRET_KEY=你的SecretKey
# COS_CDN_DOMAIN=cdn.你的域名     # 可选
```
桶设 **公共读** + 配 CORS。

---

## 改完 .env 后
```bash
pm2 restart vxin-server-v2 --update-env
# 验证（401 = 接口活了，之前是 503）
curl -s -o /dev/null -w "%{http_code}\n" -X POST http://127.0.0.1:3002/api/upload/credential
```

## 三个最易踩的坑
1. **没配 CORS** → 直传报错（前端提示"请检查 Bucket CORS 配置"）
2. **桶不公开可读** → 能传但图显示不出来（403）
3. **公开域名写错/带了 `https://`** → 图片链接拼错（`R2_PUBLIC_DOMAIN` 只填纯域名）

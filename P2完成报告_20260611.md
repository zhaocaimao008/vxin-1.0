## 🎯 模块详解

### 1. Redis 缓存层 (`src/utils/cache.js`)

**目的**: 降低数据库查询压力，加速热数据访问

```javascript
// 初始化和使用示例
const cache = require('./cache.js');
await cache.init();

// 缓存对话列表（TTL: 5 分钟）
await cache.set(
  cache.keys.conversations(userId),
  conversationList,
  300
);

// 从缓存获取
const cached = await cache.get(cache.keys.conversations(userId));

// 用户登出时清除用户相关缓存
await cache.delPattern(`user:${userId}:*`);
```

**支持的缓存键**:
- `conversations:${userId}` - 对话列表（5min）
- `user:${userId}` - 用户信息（30min）
- `search:${userId}:${query}` - 搜索结果（10min）
- `sessions:${userId}` - 会话信息（30min）

**集成点** (需手动集成到业务服务):
```javascript
// 在 conversation.service.js listConversations() 中
const cacheKey = cache.keys.conversations(userId);
let conversations = await cache.get(cacheKey);
if (!conversations) {
  conversations = db.prepare(...).all();
  await cache.set(cacheKey, conversations, 300);
}
return conversations;
```

**生产级功能**:
- ✅ 连接池管理
- ✅ 自动重连
- ✅ 错误降级（Redis 不可用时）
- ✅ TTL 自动清理

### 2. Token 黑名单扩展到 Redis

**改动**: `src/utils/tokenBlacklist.js` 升级

```javascript
// 之前: 仅内存存储（单机）
// 之后: Redis 优先，内存降级

// 使用方式不变（API 兼容）
await addToBlacklist(token, expiresAt);
const blacklisted = await isBlacklisted(token);
```

**架构**:
```
Logout 请求
  ↓
addToBlacklist(token, ttl)
  ↓
Redis DB 1: blacklist:token = '1' (with TTL)
           + 内存 Set 备用
  ↓
后续认证请求
  ↓
isBlacklisted(token) [异步]
  ↓
检查 Redis 存在性 → 返回 true/false
  （若 Redis 故障 → 降级到内存检查）
```

**分布式支持**:
- 多个后端实例共享黑名单
- Redis 过期键自动删除（无需应用干预）
- 降级机制确保服务可用性
